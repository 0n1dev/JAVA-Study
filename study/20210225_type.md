# 2. 자바 데이터 타입, 변수 그리고 배열
---

> 자바의 데이터 타입은 Primitive Type과 Reference Type으로 구분

## 1. 프리미티브 타입과 레퍼런스 타입 그리고 값의 범위와 기본 값
---

### Primitive Type
---

### 특징

- 기본값이 있기 때문에 Null이 존재하지 않음, 기본형 타입에 Null을 넣고 싶으면 레퍼 클래스 활용
- 실제 값을 저장하는 공간으로 스택 메모리에 저장
- 데이터 할당 메모리 크기를 벗어나면 에러 발생

### 종류 (타입, 할당 메모리 크기, 기본값, 데이터 표현 범위)

#### 논리형

- boolean 1byte false true,flase

#### 정수형

- byte 1byte 0 -128~-127
- short 2byte 0 -32,768~32,767
- int(기본) 4byte 0 -2,147,483,648~2,147,483,647
- long 8byte 0L -9,223,372,036,854,775,808~9,223,372,036,854,775,807

#### 실수형

- float 4byte 0.0F (3.4 X 10-38)~(3.4 X 1038) 의 근사값
- double(기본) 8byte 0.0 (1.7 X 10-308)~(1.7 X 10308) 의 근사값

#### 문자형

- char 2byte(유니코드) '\u0000' 0~65,535

### Reference Type
---

### 특징

- 기본형을 제외한 모든 타입
- 빈 객체를 의미하는 Null이 존재
- 값이 저장되어 있는 곳의 주소 값 저장 힙 메모리에 저장
- 문법상 에러는 없지만 런타임 에러가 발생

### 종류 (기본 값은 모두 Null, 메모리 크기는 4Byte(주소 값))

- Array
- Enumeration
- Class
- Interface

## 2. 리터럴
---

> 리터럴(literal)은 프로그램에서 직접 표현한 값 즉 실제로 저장되는 값을 뜻한다.

### 종류

#### 정수 리터럴

> 2진수, 8진수, 10진수, 16진수, long 타입 리터럴

- 2진수 : int v1 = 0b10;
- 8진수 : int v2 = 010;
- 10진수 : int v3 = 10;
- 16진수 : int v4 = 0x10;
- long : long v5 = 10L;

#### 실수 리터럴

- float : float v1 = 1.234F;
- double : double v1 = 1.234;
- double : double v2 = 1.234d;
- double : double v3 = 1234E - 3d;

#### 문자 리터럴

- char : char v1 = 'C';
- char : char v2 = '문';
- char : char v3 = '\u0000';

#### 논리 리터럴

- boolean : boolean v1 = true;
- boolean : boolean v2 = 1 > 2; // 조건 식

#### 문자열 리터럴

- String : String v1 = "Hello Java";

## 3. 변수 선언 및 초기화하는 방법
---

```java
int a; // 변수 선언
int b, c, d; // 여러 변수 선언 방법

a = 13; // 변수 초기화 방법

int e = 15; // 변수 선언과 초기화를 동시에
```

## 4. 변수의 스코프와 라이프타임
---

> 변수의 스코프는 해당 변수의 접근 가능한 범위 (Java에서는 블록 스코프)

```java
public class Test {

    static int a; // Test 클래스 범위

    public static void main(String[] args) {
        int b; // main 메서드 범위
    }
}
```

### 라이프 타임

- 인스턴스 변수 : 클래스를 인스턴스화한 객체가 메모리에서 사라질 때 까지
- 클래스 변수 : 프로그램 종료시 까지
- 지역 변수 : 변수가 선언된 블록이 동작하는 동안 까지

## 5. 타입 변환, 캐스팅 그리고 타입 프로모션
---

### 타입 변환

자바에는 두가지 타입 변환이 있다.

- 자동 타입 변환(묵시적)
- 강제 타입 변환(명시적)

#### 자동 타입 변환 (Promotion)

프로그램 실행 중 작은 크기의 타입이 큰 크기의 타입에 저장될 때 발생

> byte < short < int < long < float < double

#### 강제 타입 변환 (Casting)

큰 크기 타입은 작은 타입으로 자동 변환이 불가능하다. 하지만 강제로 잘라서 저장할 수 있다.

```java
int a = 111111;
byte byteA = (byte)a; // byte 크기 만큼만 저장 나머지는 버려짐
````

강제로 잘라서 저장하기 때문에 데이터의 손실이나 변형이 발생할 수 있다.

## 6. 1차 및 2차 배열 선언하기
---

배열이란 동일한 타입의 데이터를 연속된 공간에 저장하기 위한 자료구조

```java
// 1차 배열

int[] a = new int[3];

a[0] = 1;
a[1] = 2;

// or

int[] a = {1, 2};
```

```java
// 2차 배열

int[][] a = new int[2][2];

a[0][0] = 1;
a[0][1] = 2;
a[1][0] = 3;

// or

int[][] a = {{1,2}, {3,4}};
```

## 7. 타입 추론, var
---

타입 추론이란 데이터 타입을 코드에 명시하지 않아도, 컴파일 단계에서 컴파일러가 타입을 유추해 정해주는 것.

```java
var a = "TEST"; // String으로 추론
var b = 11; /// int로 추론
```
